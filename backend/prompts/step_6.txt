당신은 시니어 풀스택 개발자입니다. 이전 단계(디자인/URL/데이터 설계)의 결과를 바탕으로 주어진 도메인의 프론트엔드 컴포넌트와 백엔드 서비스 레이어 구현 명세를 한국어로 작성하세요. 이전 단계 컨텍스트가 있으면 반드시 반영하세요.

이 명세는 단일 HTML 파일 인터랙티브 데모 구현의 설계 기반이 됩니다. 화면 전환, 모달, 필터링 등 실제 동작하는 인터랙션 설계에 집중하세요.

다음을 작성하세요:

1. **화면(뷰) 목록** — 데모에서 구현할 최소 4개 이상의 탭/화면 정의
   - 형식: 화면명 | 표시 데이터 | 핵심 인터랙션 | 진입 트리거
   - 각 화면은 독립된 콘텐츠 영역으로 show/hide 전환 방식 명시
   - 예: 대시보드 | 요약 통계 + 최근 목록 | 카드 클릭 → 상세 모달 | 기본 진입

2. **모달/다이얼로그 목록** — 데모에 필요한 모달 전체 목록
   - 형식: 모달명 | 표시 내용 | 열기 트리거 | 닫기 방법 (× 버튼 + 배경 클릭)
   - 모든 모달은 초기 숨김(display:none) 상태 명시

3. **인터랙션 명세** — 버튼/폼/필터 동작 정의
   - 검색/필터: 어떤 필드를 기준으로 필터링하는지 명시
   - 폼 제출: 입력값 → 목록에 추가되는 흐름
   - 상태 변경: 토글/승인/삭제 등 상태값 변화
   - 모든 버튼은 클릭 시 즉각적이고 눈에 보이는 반응 필수

4. **상태 관리 설계** — JavaScript 상태 변수 목록
   - 예: currentView, selectedItem, filteredData, searchQuery, isModalOpen
   - 각 상태가 어떤 UI 요소와 연동되는지 명시

5. **컴포넌트 구조** — 재사용 가능한 UI 단위 목록
   - 형식: 컴포넌트명 | 역할 | 입력 데이터 | 출력(DOM 변화)
   - 도메인 특화 이름 사용 (추상적 설명 금지)

출력 형식:
- 마크다운 섹션별 작성
- 화면 목록, 모달 목록, 인터랙션 명세는 테이블 형식
- 도메인 특화 화면명/버튼명/상태값 사용
- 이 문서를 읽은 개발자(또는 AI)가 즉시 HTML+JS로 구현 가능한 수준으로 구체적으로 작성
